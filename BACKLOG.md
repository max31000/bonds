## P1 — полевой фидбек владельца (2026-07-10, после деплоя волн A/B)

### 3. БАГ: бумаги с офертой ложно классифицируются как флоатеры (подтверждено)
Владелец поймал на Брус 2Р06 (`RU000A10EC63`): фикс-купон 22.75%, BONDTYPE «Биржевая
облигация», но оферта 2028-02-18 → 12 купонов после оферты в MOEX bondization не определены
(ставка пересматривается на оферте — норма для put-оферт). `BondSyncService:393`
(`Coupons.Any(c => !c.IsKnown) → CouponType.Floating`) помечает такую бумагу флоатером.
Каскад: 422 в сравнивалке для цели; собственные позиции с офертой — «вне сравнения» в слабых
звеньях, вне матрицы/RV-вердиктов и (после T-31) вне аллокации по портфелю. Движок умеет
доходность к оферте (`CalculatedToOffer`). Правильное правило: неизвестные купоны ТОЛЬКО после
ближайшей оферты ≠ флоатер (все купоны до оферты включительно известны → Fixed, расчёт к
оферте). Неизвестные купоны ДО ближайшей оферты либо без оферты — по-прежнему Floating.
Затронутым инструментам нужен ре-синк CouponType после фикса.

### 4. Светофор надёжности из бесплатных сигналов (запрос владельца)
Платных рейтингов нет; нужен агрегированный «светофор» + фильтр по нему. Компоновать из
уже имеющегося и бесплатного: G-спред к медиане корзины (есть, T-33), уровень листинга (есть),
ликвидность (есть), сектор гос/муни/корп (есть); исследовать бесплатные прокси: ломбардный
список ЦБ (публичный), членство в индексах MOEX (RUCBITR и др. — бесплатно через ISS),
ручной импорт рейтингов АКРА/Эксперт РА для своих эмитентов (пресс-релизы публичны).
Поглощает/уточняет A4. Светофор — с дисклеймером «оценка, не рейтинг агентств».

### 5. UX подбора замен: карточка выгоды появляется на дне списка
Клик по кандидату в окне замены → карточка выгоды рендерится под списком из ~20 строк,
выглядит как «ничего не произошло»/баг. Фикс: скролл к карточке / рендер сразу под выбранной
строкой / модал-развёртка. Низкая трудоёмкость.

### 6. Подбор замен: фильтр по дюрации + дата оферты в кандидатах
В списках кандидатов (`replacement-candidates`, сравнивалка) не хватает фильтра по дюрации
(окно вокруг дюрации заменяемой позиции; на бэке `durationYears` уже есть) и колонки/пометки
даты оферты, если есть (`OfferDate` в банке есть, в DTO кандидата — нет, добавить).

## P2

### 3. Квазивалютные облигации — полноценная поддержка
Сейчас `IsOutOfScopeCurrency` выкидывает их из live-контура/метрик. Нужно: курс ЦБ/MOEX для
пересчёта номинала в ₽, метрики движка в валюте номинала + рублёвый эквивалент, отдельная
пометка в UI. По оценке — не сложно: FaceValue уже хранится в валюте, нужен курс и конверсия
на границах.

## Фичи из брейншторма 2026-07 (не взятые в первые волны)

- **C2. Пресеты-стратегии конструктора**: «сохранить дюрацию», «лесенка», «под ключевую ставку»

  P1-1/P1-2. Конструктор (задача 29) структурно уже готовит корзину.
- **D2. Алерты по банку**: сохранённый фильтр скринера + Telegram-сигнал при появлении бумаги.
- **D3. Календарь первичных размещений** (данные MOEX).
- **D4. Бенчмарк портфеля**: XIRR vs RGBITR/RUCBITR/депозитная ставка на дашборде.

## Техдолг (из ревью/аудитов, не блокирует)

- `SignalsEngine.FloaterRateResetRule` (`SignalsEngine.cs:162-190`): ложный Info-сигнал
  «пересчёт ставки купона флоатера» для фикс-бумаг с офертой — ищет ближайший `!IsKnown` купон
  без проверки `CouponType` и без среза по горизонту оферты. Тот же класс бага, что T-36
  (ревью T-36); применить правило `HasFloatingCoupon`/срез по оферте.
- `BondSyncService`: 4 старых места `DateOnly.FromDateTime(DateTime.UtcNow)` → перевести на
  `BusinessClock.MoscowToday()` (контракт `BusinessClock` — «все расчётные пути берут дату
  отсюда»); новое место T-36 уже переведено (ревью T-36).
- Тесты панели замен (ревью T-37): (а) выбранная строка выпадает при включении фильтра дюрации —
  карточка закрывается (по коду верно, теста нет); (б) позиция без `modifiedDuration` (не флоатер)
  + включённый фильтр → «0 из N»; (в) два абсолютных `offerDate`-литерала в msw-фикстурах →
  относительные даты (правило гайда).
- Бейджи «в watchlist» и «оферта {дата}» — оба grape/outline, при одновременном показе сливаются;
  развести цвет/акцент (ревью T-37).

- `GET /api/universe`: фильтрация/сортировка in-memory по ~3400 строк на каждый запрос + N+1
  по инструментам в `ResolvePortfolioIsinsAsync` — перенести в SQL при росте трафика/каталога.
- Кэш отсортированных спредов корзин в `RelativeValueService` (сейчас O(n²) на портфель — при
  текущих объёмах ок).
- N+1 по инструментам/расписаниям в `PortfolioHoldingsBuilder` (pre-existing, этап 08).
- Бэкфилл XIRR: амортизация учитывается текущим номиналом (дисклеймер есть; точный учёт — по
  графикам амортизаций).
- P-4 аудита: «купоны получено» и cost basis — до НДФЛ на купоны (в модели нет типа «налог на
  купон»); пересмотреть перед «действиями с портфелем».
- Лот = 1 бумага в аллокации/конструкторе (поля лота нет в модели; для корп. облигаций верно
  почти всегда).
- Фронт: чанк >500kB (code-splitting); `act()`-варнинги SegmentedControl; список секторов
  скринера захардкожен (зеркалит `BondUniverseSectorMapper`).
- Живой визуальный прогон 375px/тёмной темы (за Telegram-JWT — только на стенде владельца).
- Авто-оценка комиссии включает сервисные сборы (сырой тип T-Invest-операции не хранится) —
  при необходимости точности хранить сырой тип.
- `bond_universe`: нет soft-delete выведенных с торгов бумаг (отсеиваются фильтром косвенно);
  для длинной RV-истории может понадобиться явный признак.
- Скринер, фильтр «только фикс-купон» (T-32, ревью волны A): клиентский фильтр поверх серверной
  пагинации — счётчик «N бумаг · страница X из Y» показывает нефильтрованный total, страница может
  быть короче PAGE_SIZE. Правильный фикс — серверный параметр типа купона в `GET /api/universe`.
- Скринер: empty-state «Ничего не найдено» гейтится на `rows.length`, а не `displayedRows.length` —
  если на странице все бумаги флоатеры и фильтр включён, таблица пустая без объяснения
  (`Screener.tsx`, ревью волны A).
- `MarketComparator`: выпадашка целей — `limit 20 → фильтр флоатеров → top-10`; если флоатеров в
  топе доходности >10, опций останется мало без индикации усечения. Поднять limit или показать
  «показаны доступные N» (ревью волны A).
- Порог спред-сигнала продублирован: `CandidateRiskSignalService.SpreadDeviationThresholdFraction`
  (Core) и `AnalyticsEndpoints.FairVerdictThresholdFraction` (Api), оба 0.0020 — вынести в общую
  Core-константу, чтобы не разъехались при пересмотре (ревью волны B).
- Мобильная вёрстка новых блоков «Рекомендаций» (T-35) не проверена явно на 375px — код на
  адаптивных примитивах (`wrap`, ScrollContainer), но без `useMediaQuery`/теста; покрыть при
  ближайшем живом прогоне мобильной темы (ревью волны B; связано с существующим пунктом про
  визуальный прогон 375px).
