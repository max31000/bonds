# Bond Portfolio Analytics — продуктовая спецификация (handoff для агента-имплементатора)

> **Назначение документа.** Это бизнес-описание продукта и его составляющих. Документ предназначен для передачи AI-агенту, имеющему доступ к репозиториям существующих сервисов. Агент использует этот файл вместе с кодом репозиториев, чтобы собрать технический план реализации (разбивка на модули, контракты, схемы БД, последовательность работ). Документ намеренно описывает **что** и **зачем**, оставляя **как** на усмотрение агента и фактической структуры репозиториев. Указания специально для имплементатора собраны в разделе 12.

---

## 1. Краткое описание

Персональный аналитический сервис для облигационного портфеля. Подтягивает реальные позиции и операции с брокерского счёта (Т-Инвестиции) и обогащает их полноценной облигационной математикой и календарём денежного потока, которых нет в приложении брокера и в обычных портфельных трекерах.

Продукт закрывает зазор между двумя классами существующих решений:

- **Портфельные трекеры** (Intelinvest, Snowball Income, izi-invest) — знают твой портфель и хорошо ведут учёт, но к облигациям относятся как к обобщённому источнику кэш-флоу: не считают корректный YTM по текущей цене, дюрацию, спреды, не моделируют амортизацию/оферты, путаются в НКД и флоатерах.
- **Облигационная аналитика** (dohod.ru, RusBonds, Cbonds) — считают глубокую математику долга, но это скринеры по всей вселенной бумаг, не привязанные к твоим позициям, твоей цене покупки и твоему налогу.

Продукт = **portfolio-aware bond analytics**: глубокая облигационная математика, применённая к реальным позициям, с персональным календарём поступлений, налогом и сигналами.

## 2. Целевой пользователь и режим использования

- Один пользователь (владелец), один брокерский счёт. Не мультитенантный продукт на MVP.
- Портфель из единиц–десятков облигационных позиций.
- Деплой в существующую облачную инфраструктуру пользователя (фронт + бэк + домен уже работают).
- Дневной режим: данные обновляются батчем, не в реальном времени (для облигаций этого достаточно).

## 3. Границы продукта (MVP)

### В скоупе
- Синхронизация позиций и операций с одного брокерского счёта Т-Инвестиций.
- Справочник облигаций: графики купонов, амортизаций, оферт.
- Расчётные метрики: НКД, грязная цена, YTM (эффективная и простая), доходность к оферте, дюрация Маколея, модифицированная дюрация, выпуклость, PVBP, G-спред к безрисковой кривой.
- Корректная обработка флоатеров (YTM неприменим — показывать доходность по текущей ставке с явной пометкой) и амортизируемых бумаг (убывающий номинал в денежном потоке).
- Персональный календарь денежного потока (купоны, амортизации, погашения) с учётом НДФЛ 13%.
- Доходность портфеля (XIRR по журналу операций).
- Сортировка позиций по доходности (со встроенным дисклеймером про риск) — простой режим сравнения.
- Анализ замены в рамках **текущих** позиций.
- Триггеры/сигналы (см. раздел 8).
- Визуальный слой (см. раздел 9).
- Синхронизация 1–2 раза в первые часы торгов MOEX + кнопка форс-обновления в UI.

### Вне скоупа (будущие фичи, не закладывать как обязательную зависимость)
- **Кредитные рейтинги** в любом виде — полностью вне MVP (автоматический приём рейтингов требует платного data feed агентств).
- Планирование вывода средств / оптимизация момента ликвидации.
- Скринер по всей вселенной облигаций (поиск кандидатов на замену вне текущего портфеля).
- Несколько счетов / ИИС с раздельной налоговой логикой.
- Полноценное налоговое моделирование (сальдирование убытков, налог при продаже в анализе замены) — на MVP только плоский НДФЛ 13% на купоны.
- AI-слой (принятие микрорешений, неформализуемые автоматизации) — навешивается отдельно поверх посчитанных движком метрик.

## 4. Источники данных и архитектура данных

**Принцип: только бесплатные API, ни один модуль MVP не должен зависеть от платного источника.**

### 4.1. T-Invest API (Т-Инвестиции) — «истина про счёт»
Бесплатный, доступ по токену. Рекомендуется **read-only токен** (исполнение сделок в продукте отсутствует).

Поставляет:
- Позиции портфеля с количеством и средневзвешенной ценой покупки (cost basis).
- Журнал операций: покупки, продажи, поступившие купоны, амортизационные выплаты, удержанный налог, комиссии. Источник истины для XIRR.
- Базовые параметры инструмента и текущие цены.
- График купонов.

Чего T-Invest по облигациям **не отдаёт надёжно** и что берётся из MOEX: полный график амортизации (обычно только флаг), расписание оферт, корректный аналитический референс (YTM/дюрация/кривая).

### 4.2. MOEX ISS — справочно-аналитический слой
Бесплатный, без регистрации, без заявленных лимитов на вызовы. Формат JSON/XML.

| Данные | Эндпоинт (шаблон) | Назначение |
|---|---|---|
| Купоны + амортизации + оферты | `/iss/statistics/engines/stock/markets/bonds/bondization/{SECID}.json?iss.only=coupons,amortizations,offers` | Денежный поток, сигналы по офертам |
| Параметры выпуска + цены | `/iss/engines/stock/markets/bonds/securities/{SECID}.json` | Дата погашения, дата оферты, купон (%/руб), номинал, доходность |
| Рыночные данные / стакан | marketdata / orderbook на соответствующем board | Текущая чистая цена, ликвидность |
| Кривая Gcurve (КБД) | `/iss/engines/stock/zcyc/securities.json` | Параметры Нельсона-Сигеля-Свенссона (B1,B2,B3,T1,G1…G9) для реконструкции безрисковой кривой → G-спред |
| Архив Gcurve | Gcurve на закрытие дня (архив) | Историческая кривая для ретроспективы |

`{SECID}` — биржевой идентификатор бумаги; нужен маппинг из идентификаторов T-Invest (FIGI/ISIN) в SECID MOEX. ISIN — общий ключ для маппинга.

### 4.3. Платные зоны и юридические пометки
- **Рейтинги** — структурированная выгрузка платная у всех агентств (АКРА, Эксперт РА, НКР, НРА) и у агрегаторов (Cbonds/Investfunds). Вне MVP целиком.
- «КБД Московской Биржи» / «MOEX GCURVE» — зарегистрированный товарный знак. Сами данные использовать можно; брендированное **название** в публичном UI — нельзя (для личного использования неактуально; при публичном релизе — переименовать, например в «безрисковую кривую»).
- Данные ISS — собственность MOEX. Личное использование — ок; публичное перераспределение требует проверки Terms of Use.

### 4.4. Риски полноты данных (обязательно заложить в реализацию)
- **Неполные купоны в `bondization`.** По части выпусков MOEX отдаёт не все купоны. Обрабатывать пропуски: помечать позицию как «данные неполные», не падать, не подставлять нули молча.
- **Флоатеры.** Будущие купоны неизвестны принципиально — YTM не считается. Показывать доходность по текущей ставке / к ближайшему пересчёту, помечать визуально как плавающую.
- **Амортизация.** Денежный поток и средневзвешенная цена должны учитывать убывающий номинал; типичная ошибка — считать как обычную бумагу.
- **Оферта vs погашение.** Для бумаг с put/call расчёт «к погашению» некорректен — считать к ближайшей неисполненной оферте (с отсечкой по сроку, см. 7.3).
- **Контракт T-Invest.** Точную схему полей по облигациям (наличие амортизации/оферт в ответе) имплементатор сверяет с актуальным gRPC/REST-контрактом, а не принимает на веру; источник истины по расписаниям — в любом случае MOEX.

## 5. Доменная модель (сущности и абстракции)

Ключевой разрез: разделить **справочные данные инструмента** (общие для всех), **рыночные котировки** (временной ряд) и **позицию** (cost basis, операции пользователя).

- **Instrument** — справочник выпуска: ISIN, SECID (MOEX), FIGI (T-Invest), эмитент, сектор, номинал, валюта, тип купона (фиксированный / плавающий / индексируемый), флаги амортизации и оферт, дата погашения.
- **CouponSchedule** — даты и размеры купонов. Для флоатера купон известен только до ближайшего пересчёта.
- **AmortizationSchedule** — даты и доли/суммы погашения номинала.
- **OfferSchedule** — put/call-оферты с датами.
- **MarketQuote** — временной ряд: чистая цена, грязная цена, НКД, объём торгов (ликвидность), снимок расчётных метрик на дату.
- **YieldCurveSnapshot** — параметры Gcurve на дату (для G-спреда и истории).
- **Position** — холдинг: инструмент, количество, средневзвешенная цена покупки, текущий накопленный НКД, привязка к счёту.
- **Operation** — событие журнала: покупка, продажа, купон, амортизация, погашение, налог, комиссия. Источник истины для XIRR.
- **ProjectedCashFlow** — производная сущность: спроецированные будущие поступления (купон/амортизация/погашение) с датой, суммой брутто, удержанным НДФЛ, суммой нетто и признаком определённости (для флоатеров — «оценка»).
- **Portfolio / Account** — агрегат (на MVP один брокерский счёт).
- **TargetAllocation** (опционально на MVP) — целевые доли и лимиты концентрации для триггеров и ребаланса.
- **Signal** — сгенерированное событие-триггер: тип, важность, привязка к позиции, предлагаемое действие, дата.

## 6. Движок расчётов (метрики)

Все расчёты — аналитические оценки, **не** инвестиционные рекомендации (зафиксировать дисклеймер в UI, по аналогии с дисклеймером калькулятора MOEX).

По нарастающей сложности:

1. **НКД** — пропорциональное накопление между датами купонов.
2. **Грязная цена** = чистая цена + НКД.
3. **YTM по текущей цене** — решение IRR: ставка, при которой приведённая стоимость будущего денежного потока равна текущей грязной цене. Считать эффективную (годовое сложное начисление) и простую. Метод — Ньютон-Рафсон с фолбэком на бисекцию (гарантия сходимости).
4. **Доходность к оферте** — тот же расчёт, но горизонт до ближайшей неисполненной оферты (см. 7.3).
5. **Дюрация Маколея** — средневзвешенный по приведённой стоимости срок поступлений.
6. **Модифицированная дюрация** = Маколей / (1 + y/k), где k — число купонных периодов в году. Мера чувствительности цены к ставкам.
7. **Выпуклость** и **PVBP** (= мод. дюрация × грязная цена × 0.0001) — для оценки изменения цены при сдвиге ставок.
8. **G-спред** = YTM бумаги − значение Gcurve, интерполированное на срок/дюрацию бумаги. База для честного сравнения «лучше/хуже» при сопоставимом сроке. (Z-спред к полной нулевой кривой — опционально, сложнее; на MVP достаточно G-спреда.)
9. **XIRR портфеля** — внутренняя норма доходности по всему журналу операций (покупки со знаком минус, купоны/продажи плюс, текущая стоимость как терминальный поток).

### Краевые случаи (обязательны)
- **Флоатер:** YTM не считать. Выводить текущую купонную доходность (текущий купон, приведённый к году, к грязной цене) и доходность к ближайшему пересчёту/оферте. Метка «плавающая, может измениться».
- **Амортизируемая бумага:** денежный поток включает частичные возвраты номинала; купон начисляется на остаточный номинал; средневзвешенная цена и метрики учитывают убывающее тело.
- **Бумага с офертой:** все метрики — к ближайшей оферте, не к погашению.
- **Неполные данные:** метрика помечается как недостоверная, а не подставляется молча.

## 7. Денежный поток (календарь поступлений)

7.1. Детерминированная проекция по каждой позиции: купоны × количество, амортизационные выплаты, погашение тела. Для флоатеров — оценка по текущей ставке с пометкой.

7.2. Удержание **НДФЛ 13%** на купонный доход; в календаре показывать брутто, налог и нетто. (Амортизация/возврат номинала налогом на купон не облагается — разделять типы потоков.)

7.3. **Отсечка по оферте:** при расчёте «к оферте» выбирается ближайшая неисполненная put/call-оферта, до которой остаётся не менее ~14 календарных дней (конвенция, как в профильных калькуляторах); учитываются только потоки до неё.

7.4. Агрегация по месяцам и по позициям для виджета-календаря; отдельно — даты освобождения тела (погашения/крупные амортизации) для понимания, когда появится ликвидность.

## 8. Триггеры и сигналы (событийный слой)

Каждый триггер — правило + настраиваемый параметр; генерирует `Signal`.

- Приближается купон / амортизация / погашение (за N дней).
- **Приближается оферта (put/call)** — приоритетный сигнал: put-оферту нужно активно подавать через брокера, иначе бумага остаётся в портфеле. Легко пропустить — высокая ценность.
- Пересчёт купона у флоатера / смена ставки.
- Накопился незаинвестированный кэш (купоны) выше порога — напоминание о реинвесте.
- Доходность удерживаемой бумаги ушла ниже сопоставимой по сроку альтернативы из портфеля более чем на X базисных пунктов.
- Нарушение лимита концентрации по эмитенту (порог настраивается) — защита от риска единичного эмитента.
- Дрейф дюрации портфеля от целевой (если задана `TargetAllocation`).
- Предупреждение о низкой ликвидности по позиции (тонкий стакан) — на будущее, при появлении планирования вывода.

(Сигнал по снижению рейтинга — вне MVP, т.к. рейтинги вынесены за скоуп.)

## 9. Аналитика, сравнение и визуальный слой

### Сравнение позиций (MVP — простой режим)
Сортировка позиций по доходности с явным дисклеймером: более низкая доходность ≠ «хуже» без учёта срока и риска. Колонки таблицы: расчётный YTM (или текущая доходность для флоатера), дюрация, G-спред, дни до погашения/оферты, тип купона. Анализ замены — только между текущими позициями, с учётом комиссии обеих сделок (налог при продаже — вне MVP).

### Визуализация (графика — приоритет продукта)
- **Календарь поступлений** — таймлайн/бары по месяцам (брутто/налог/нетто).
- **Scatter «дюрация × доходность»** твоих бумаг поверх кривой Gcurve — визуально видно, кто над/под кривой.
- **Композиция портфеля** — по эмитенту, сектору, типу купона, корзинам дюрации.
- **Таблица позиций** с расчётными метриками и сортировкой.
- **Кривая доходности портфеля во времени** (XIRR/динамика стоимости).

## 10. Функциональные модули (декомпозиция для техплана)

Логические модули; имплементатор соотносит их с фактическими сервисами/репозиториями и решает, что переиспользовать, а что создать.

1. **Broker Sync (T-Invest connector)** — синхронизация позиций и журнала операций по read-only токену.
2. **Reference Data (MOEX ISS connector)** — справочник выпусков, графики купонов/амортизаций/оферт, маппинг ISIN→SECID, снимки Gcurve. Кэширование справочных данных (меняются редко).
3. **Storage** — хранилища: справочные данные, котировки (временной ряд), позиции и операции (истина про счёт), сигналы.
4. **Calculation Engine** — НКД, грязная цена, YTM/доходность к оферте, дюрации, выпуклость, PVBP, G-спред; обработка флоатеров и амортизации.
5. **Cash-Flow Projection** — проекция поступлений с НДФЛ, агрегация по месяцам/позициям.
6. **Portfolio Analytics** — XIRR, композиция, сравнение/сортировка позиций, анализ замены.
7. **Signals Engine** — вычисление триггеров по расписанию и по событиям обновления данных.
8. **Scheduler** — окна автосинка в первые часы торгов MOEX + обработка форс-обновления из UI.
9. **Backend API** — контракты для фронта.
10. **Frontend** — дашборды и виджеты (раздел 9).

## 11. Нефункциональные требования

- **Бесплатность зависимостей:** только T-Invest API + MOEX ISS. Никаких платных источников в обязательном контуре.
- **Синхронизация:** дневной батч, 1–2 запуска в первые часы торгов MOEX; ручной форс-апдейт из UI. Realtime не требуется.
- **Безопасность:** брокерский токен — read-only, хранить как секрет (исполнение сделок в продукте отсутствует). Токен не логировать, не передавать на фронт.
- **Валюта:** RUB как базовая (портфель рублёвый).
- **Устойчивость к неполным данным:** деградация с пометками вместо падения (см. 4.4).
- **Дисклеймер:** все расчёты — аналитические оценки, не инвестиционные рекомендации.

## 12. Указания для агента-имплементатора

1. **Сверь модули с репозиториями.** Прежде чем планировать, изучи существующие сервисы: какие из модулей раздела 10 уже частично закрыты (HTTP-клиенты, планировщики, хранилища, фронтовые компоненты графиков) и могут быть переиспользованы. Предложи маппинг «модуль → репозиторий/сервис».
2. **Верифицируй контракт T-Invest.** Точные имена методов и поля по облигациям (амортизация, оферты, график купонов) подтверди по актуальному gRPC/REST-контракту. Источник истины по расписаниям — MOEX ISS, даже если T-Invest что-то дублирует.
3. **Начни с вертикального среза Tier 0→1.** Сначала сквозной путь: синк позиций → справочник по MOEX → НКД/грязная цена → календарь поступлений → базовая таблица. Затем наращивай YTM/дюрацию/G-спред и сигналы.
4. **Изолируй Calculation Engine** как чистый, хорошо покрытый тестами модуль (детерминированная математика, краевые случаи из 6 и 7 — обязательные тест-кейсы: флоатер, амортизация, оферта, неполные купоны).
5. **Заложи точки расширения** под вынесенные за MVP фичи (рейтинги, мультисчёт, планирование вывода, скринер по вселенной, AI-слой), но не реализуй их.
6. **Выходной артефакт:** технический план с разбивкой по модулям, контрактами API, схемами хранения, последовательностью работ и тест-стратегией.

## 13. Глоссарий

- **НКД** — накопленный купонный доход: часть купона, накопленная с последней выплаты; платится продавцу при покупке.
- **Грязная / чистая цена** — с НКД / без НКД.
- **YTM (доходность к погашению)** — ставка, уравнивающая приведённую стоимость будущих потоков с текущей грязной ценой.
- **Дюрация Маколея** — средневзвешенный срок поступлений; **модифицированная дюрация** — чувствительность цены к ставкам.
- **PVBP / выпуклость** — оценки изменения цены при сдвиге ставок.
- **Оферта (put/call)** — право досрочного предъявления/отзыва бумаги в заданную дату; put требует активного действия инвестора.
- **Амортизация** — поэтапный возврат номинала в течение срока обращения.
- **Флоатер** — облигация с плавающим купоном; будущие купоны заранее неизвестны, YTM не определён.
- **G-спред** — премия доходности бумаги над безрисковой кривой на сопоставимом сроке.
- **Gcurve (КБД)** — кривая бескупонной доходности по госбумагам; безрисковый эталон.
