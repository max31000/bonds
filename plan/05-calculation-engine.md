# Этап 05 — Расчётный движок (Calculation Engine)

> **Зависит от:** этапы 03–04 (модели + данные).
> **Опора по требованиям:** §6 (метрики + краевые случаи), §7.3 (отсечка по оферте), §4.4, §13 (глоссарий). §12.4 — этот модуль изолирован и максимально покрыт тестами.
> **Эталон изоляции:** в cashpulse `Core/Forecast/ForecastEngine.cs` — чистая детерминированная логика без I/O, отдельные юнит-тесты.

## Цель этапа

Реализовать **чистый, детерминированный, без I/O** расчётный движок облигационной математики в `Bonds.Core/Calculation`. На вход — данные инструмента/расписаний/котировок/кривой (из этапа 03–04), на выход — метрики. Никаких обращений к БД/сети внутри движка (всё передаётся аргументами) — это делает его полностью юнит-тестируемым.

> Все метрики — аналитические оценки, не рекомендации (§6). Краевые случаи (флоатер, амортизация, оферта, неполные данные) — **обязательные** тест-кейсы, а не опция (§6, §12.4).

---

## Часть A — Метрики (по нарастающей сложности, §6)

Реализовать как чистые функции/сервисы (предложение по разбивке на файлы).

> **Входные цена и НКД.** Для открытых позиций движок получает **текущую цену и НКД из T-Invest** (этап 04, `market_quotes` с `Source=TInvest`) — это первичные входы для грязной цены/YTM. Собственный `AccruedInterestCalculator` используется как fallback (бумаги без позиции — кандидаты на замену) и для сверки. Движок остаётся чистым: значения передаются аргументами, он не ходит в БД/сеть.

1. **НКД** (`AccruedInterestCalculator`) — пропорциональное накопление между датами купонов (§6.1, §13). Fallback, когда НКД из T-Invest недоступен.
2. **Грязная цена** = чистая + НКД (§6.2).
3. **YTM по текущей цене** (`YtmCalculator`) — решение IRR: ставка, уравнивающая PV будущего денежного потока с текущей грязной ценой. Считать **эффективную** (годовое сложное) и **простую**. Метод — **Ньютон-Рафсон с фолбэком на бисекцию** (гарантия сходимости) (§6.3). Денежный поток учитывает амортизацию (убывающий номинал).
4. **Доходность к оферте** (`YieldToOfferCalculator`) — тот же расчёт, горизонт до ближайшей **неисполненной** оферты с отсечкой ≥ ~14 календарных дней (§6.4, §7.3).
5. **Дюрация Маколея** (`DurationCalculator`) — средневзвешенный по PV срок поступлений (§6.5).
6. **Модифицированная дюрация** = Маколей / (1 + y/k), k — число купонных периодов в году (§6.6).
7. **Выпуклость** и **PVBP** (= мод. дюрация × грязная цена × 0.0001) (§6.7).
8. **G-спред** (`GSpreadCalculator`) = YTM бумаги − значение Gcurve, интерполированное на срок/дюрацию бумаги. Реконструкция безрисковой кривой из параметров Нельсона-Сигеля-Свенссона (`YieldCurveSnapshot`), интерполяция на нужный срок (§6.8). (Z-спред — вне MVP.)
9. **XIRR портфеля** (`XirrCalculator`) — IRR по всему журналу операций: покупки «−», купоны/продажи «+», текущая стоимость — терминальный поток (§6.9). Тоже Ньютон-Рафсон + бисекция.

## Часть B — Краевые случаи (обязательные, §6 / §4.4)

- **Флоатер:** YTM **не считать**. Возвращать текущую купонную доходность (текущий купон, приведённый к году, к грязной цене) и доходность к ближайшему пересчёту/оферте. Метрика помечается «плавающая, может измениться» (флаг в результате).
- **Индексируемая бумага (`CouponType=Indexed`, напр. ОФЗ-ИН/линкеры):** номинал/купон зависят от будущей инфляции — будущий поток принципиально неизвестен, как у флоатера. **Трактовать как флоатер-подобную:** классический YTM не считать, возвращать текущую доходность с флагом `IsEstimated` и пометкой «индексируемая, оценка». Не выдавать псевдоточный YTM (тот же класс тихих ошибок, что §4.4).
- **Амортизируемая бумага:** денежный поток включает частичные возвраты номинала; купон начисляется на **остаточный** номинал; дюрация/YTM учитывают убывающее тело.
- **Бумага с офертой:** все метрики — к ближайшей неисполненной оферте, не к погашению (отсечка §7.3).
- **Неполные данные:** метрика помечается недостоверной (флаг), а не подставляется молча. Движок не бросает исключение на неполноте — возвращает результат с пометкой.

## Часть C — Контракт результата

- Тип результата метрик (напр. `BondMetrics`) несёт сами значения **и** флаги качества/применимости: `IsFloater`, `IsIndexed`, `IsEstimated`, `DataIncomplete`, `CalculatedToOffer` (vs к погашению), `Confidence`/`Notes`. Потребители (этапы 06, 08, 09) показывают пометки в UI.
- Все методы — без побочных эффектов; вход — value-объекты/DTO, не репозитории.

## Часть D — Тесты (`Bonds.Tests`) — ядро этапа

Это самый покрытый тестами модуль. Обязательные кейсы с **эталонными значениями** (сверять с профильными калькуляторами/ручным расчётом):
- Фикс-купонная бумага: НКД, грязная цена, YTM (эффективная и простая), дюрация Маколея/мод., выпуклость, PVBP — известные эталонные числа (допуск задать явно, напр. ±1e-4 по доходности).
- **Флоатер:** YTM не считается; возвращается текущая доходность + флаг «плавающая».
- **Индексируемая:** YTM не считается; возвращается текущая доходность + флаги `IsIndexed`/`IsEstimated`.
- **Амортизация:** денежный поток с убывающим номиналом; купон на остаток; YTM/дюрация отличаются от неамортизируемого аналога предсказуемо.
- **Оферта:** расчёт к ближайшей неисполненной оферте; оферта ближе 14 дней игнорируется (берётся следующая/погашение) — проверить отсечку §7.3.
- **Неполные данные:** метрика помечается недостоверной, исключения нет.
- **YTM-сходимость:** Ньютон-Рафсон сходится; на «плохих» входах срабатывает фолбэк на бисекцию и тоже сходится.
- **G-спред:** реконструкция NSS-кривой на синтетических параметрах даёт известные значения; интерполяция на срок корректна.
- **XIRR:** на простом журнале (одна покупка + купоны + терминал) совпадает с эталоном; устойчив к нерегулярным датам.

## Часть E — Запись снимков метрик

- Предусмотреть сохранение снимка рассчитанных метрик на дату в `MarketQuote`/отдельную таблицу (для истории и scatter «дюрация×доходность» этапа 09). Сам пересчёт оркеструется этапом 07 (по расписанию) — здесь только чистые функции + контракт.

## Критерии приёмки
- [ ] Все метрики §6 реализованы как чистые функции без I/O.
- [ ] YTM и XIRR используют Ньютон-Рафсон с фолбэком на бисекцию; сходимость доказана тестами.
- [ ] Флоатер: YTM не считается, возвращается текущая доходность + флаг.
- [ ] Амортизация учитывается в денежном потоке и метриках.
- [ ] Расчёт к оферте уважает отсечку ≥14 дней (§7.3).
- [ ] Неполные данные → пометка, без исключений.
- [ ] G-спред считается через реконструкцию NSS-кривой и интерполяцию на срок.
- [ ] Все обязательные тест-кейсы (часть D) зелёные, с явными допусками против эталонных значений.
