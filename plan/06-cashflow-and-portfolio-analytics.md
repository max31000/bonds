# Этап 06 — Проекция денежного потока и портфельная аналитика

> **Зависит от:** этап 05 (движок) + этапы 03–04 (данные).
> **Опора по требованиям:** §7 (денежный поток), §9 «Сравнение позиций», §6.9 (XIRR), §3 (анализ замены в рамках текущих позиций). §10 модули 5–6.

## Цель этапа

Поверх чистого движка построить два прикладных слоя:
1. **Cash-Flow Projection** (`Bonds.Core/CashFlow`) — персональный календарь будущих поступлений с НДФЛ.
2. **Portfolio Analytics** (`Bonds.Core/Analytics`) — XIRR, композиция, сравнение/сортировка позиций, анализ замены.

После этапа эти сервисы возвращают готовые к отдаче данные (но без HTTP — это этап 08).

---

## Часть A — Cash-Flow Projection (§7)

### A1. Детерминированная проекция (§7.1)
По каждой позиции спроецировать будущие поступления: купоны × количество, амортизационные выплаты, погашение тела. Для **флоатеров и индексируемых бумаг** — оценка по текущей ставке/номиналу с пометкой `IsEstimated` (§7.1, §6 краевые случаи). Не-RUB бумаги — вне MVP (исключаются из проекции, см. `03`). Результат — записи `ProjectedCashFlow`.

### A2. НДФЛ 13% (§7.2)
- Удержание **НДФЛ 13%** на **купонный** доход; в календаре показывать брутто, налог, нетто.
- **Амортизация/возврат номинала купонным налогом не облагается** — разделять типы потоков (`FlowType`): к Coupon применяется налог, к Amortization/Redemption — нет.
- **Это проекция (оценка) будущего налога**, не факт. Фактически удержанный налог по уже прошедшим купонам приходит из журнала T-Invest (`operations`, Type=Tax, этап 04) и используется в XIRR (B1). Не смешивать: календарь = оценка 13% вперёд; XIRR = фактические налоги из журнала.

### A3. Отсечка по оферте (§7.3)
При горизонте «к оферте» — ближайшая неисполненная put/call-оферта с остатком ≥ ~14 дней; учитываются только потоки до неё. Логику отсечки переиспользовать из движка (этап 05), не дублировать.

### A4. Агрегации (§7.4)
- По месяцам и по позициям — для виджета-календаря (брутто/налог/нетто).
- Отдельно — **даты освобождения тела** (погашения/крупные амортизации) — когда появится ликвидность.

### A5. Тесты (`Bonds.Tests`)
- Амортизируемая бумага: проекция с убывающим телом; налог только на купон, не на возврат номинала.
- Флоатер: потоки помечены `IsEstimated`.
- Бумага с офертой: проекция до оферты с отсечкой §7.3.
- Агрегация по месяцам суммирует брутто/налог/нетто корректно; нетто = брутто − налог.

## Часть B — Portfolio Analytics (§6.9, §9)

### B1. XIRR портфеля и история NAV
- Текущий XIRR: `XirrCalculator` (этап 05) над полным журналом `operations` владельца (с **фактическими** налогами/комиссиями) + терминальная текущая стоимость.
- **Кривая доходности портфеля во времени** (§9) строится из таблицы `portfolio_value_snapshots` (этап 03): ряд ежедневных снимков `MarketValueRub`/`XirrToDate`. Этот сервис умеет (а) посчитать снимок «на сейчас» (его сохраняет планировщик, этап 07) и (б) отдать исторический ряд для графика. Без персистентных снимков кривую с прошлого построить нельзя — поэтому копим с первого запуска.

### B2. Композиция портфеля
Доли по: эмитенту, сектору, типу купона, корзинам дюрации (§9). Суммы долей = 100%. Источник дюрации — метрики движка.

### B3. Сравнение/сортировка позиций (простой режим, §9)
Таблица позиций с колонками: расчётный YTM (или текущая доходность для флоатера), дюрация, G-спред, дни до погашения/оферты, тип купона. Сортировка по доходности. **Обязательно** — флаг/дисклеймер: более низкая доходность ≠ «хуже» без учёта срока и риска (§9, §6).

### B4. Анализ замены (§3, §9)
Только между **текущими** позициями (не скринер по вселенной — вне MVP). Сравнение «удерживать A vs переложиться в B» с учётом **комиссии обеих сделок**. Налог при продаже — вне MVP (плоский НДФЛ только на купоны). Вернуть оценку выгоды/проигрыша с дисклеймером.

### B5. Тесты
- XIRR портфеля на эталонном журнале совпадает с ожидаемым (переиспользует тесты движка, но на портфельном уровне).
- Композиция: доли суммируются в 100% на синтетическом портфеле.
- Сравнение позиций: сортировка корректна; флоатеры показывают текущую доходность, а не YTM.
- Анализ замены: учитывает комиссии обеих сделок; не лезет за пределы текущих позиций.

## Критерии приёмки
- [ ] Проекция потока строит купоны/амортизации/погашения по позициям; флоатеры помечены `IsEstimated`.
- [ ] НДФЛ 13% применяется только к купонам; возврат номинала не облагается; в результате есть брутто/налог/нетто.
- [ ] Агрегации по месяцам/позициям и даты освобождения тела доступны.
- [ ] XIRR портфеля считается по журналу + терминал; есть ряд для кривой во времени.
- [ ] Композиция по эмитенту/сектору/типу купона/корзинам дюрации (сумма 100%).
- [ ] Сравнение позиций с колонками §9 и обязательным дисклеймером о риске; флоатеры — текущая доходность.
- [ ] Анализ замены учитывает комиссии обеих сделок и ограничен текущими позициями.
- [ ] Все тесты части A/B зелёные.
