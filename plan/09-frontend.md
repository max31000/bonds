# Этап 09 — Фронтенд: дашборды и виджеты

> **Зависит от:** этап 08 (API), этап 02 (auth-фронт).
> **Опора по требованиям:** §9 (визуальный слой — приоритет продукта), §6/§11 (дисклеймер). §10 модуль 10.
> **Эталоны:** cashpulse `cashpulse-web/` (структура `api/`, `store/`, `components/`, `pages/`, `hooks/`, `utils/`, `theme.ts`), `portal-shell/docs/service-routing-integration.md` (basename, iframe, postMessage).

## Цель этапа

Собрать UI на React 19 + Mantine 9 + Recharts 3, встраиваемый в portal-shell и работающий standalone. Графика — приоритет (§9). К концу этапа есть сквозной рабочий продукт: вход → таблица позиций → календарь → аналитика → сигналы, задеплоенный на `/bonds/`.

## Часть A — Каркас приложения

- Структура как в cashpulse: `src/{api,store,components,pages,hooks,utils,theme.ts,App.tsx,main.tsx}`.
- React Router 7, `basename = import.meta.env.BASE_URL` (`/bonds/`). Маршруты без префикса субпути.
- Vite `base: '/bonds/'` (уже из этапа 01). SPA-fallback для прямых ссылок обеспечивает nginx `try_files` (этап 01).
- Mantine 9 провайдер + тема; учесть отличия Mantine 9 (нет `Collapse in`, особенности `Grid`) — см. cashpulse `FRONTEND_DECISIONS.md`.
- Zustand-сторы (по ресурсу): `useAuthStore` (этап 02, persist), `usePositionsStore`, `useSignalsStore`, `useSettingsStore` (persist: тема, пороги).
- API-клиент `api/client.ts`: база `import.meta.env.VITE_API_BASE ?? '/bonds/api'`, заголовок `Authorization`, обработка 401 (logout→/login).
- **iframe-контекст** (`useIsInsideShell`): внутри shell скрывать собственную навигацию/«выйти» при необходимости; отправлять `postMessage` о навигации на `https://mvv42.ru` (targetOrigin строго, не `*`) — см. routing-doc.
- **Дисклеймер** (расчёты — аналитические оценки, не рекомендации) виден на ключевых экранах (§6, §11).

## Часть B — Экраны и виджеты (§9)

1. **Таблица позиций** — расчётный YTM (или текущая доходность для флоатера), дюрация, G-спред, дни до погашения/оферты, тип купона; сортировка по доходности; визуальные пометки флоатеров/неполных данных/«к оферте»; дисклеймер о риске рядом с сортировкой.
2. **Календарь поступлений** — таймлайн/бары по месяцам (брутто/налог/нетто), Recharts; отдельно даты освобождения тела (ликвидность).
3. **Scatter «дюрация × доходность»** — точки твоих бумаг поверх **«безрисковой кривой»** (реконструированная кривая Gcurve); видно, кто над/под кривой. ⚠️ В UI называть её именно «безрисковая кривая» / «безрисковая доходность», **не** брендированным «КБД Московской Биржи» / «MOEX GCURVE» — это зарегистрированный товарный знак (§4.3); сами данные использовать можно, брендированное название в UI — нет.
4. **Композиция портфеля** — по эмитенту, сектору, типу купона, корзинам дюрации (доли).
5. **Кривая доходности портфеля во времени** — XIRR/динамика стоимости (Recharts).
6. **Панель сигналов** — список с важностью (оферты — акцентно), действие, дата; пометить прочитанным/закрыть.
7. **Кнопка форс-обновления** — `POST /bonds/api/sync`, индикатор статуса (`/sync/status`), нотификация Mantine по завершении.
8. **Настройки** — пороги триггеров, `TargetAllocation`, ввод/замена токена T-Invest (`PUT /api/settings/tinvest-token`; токен не отображается обратно — только статус/маска), тема.
9. **Экран логина** (этап 02) и защита роутов.

## Часть C — Интеграция с portal-shell

- Запись `bonds` в `portal-shell/registry.json` (этап 01) — проверить, что сервис виден и грузится в iframe (`https://mvv42.ru/?app=bonds`).
- Пройти **чеклист** из `portal-shell/docs/service-routing-integration.md`: ассеты с `/bonds/assets/...`; внутренняя навигация даёт `/bonds/...`; F5 на вложенном роуте не 404; deep link работает; нет дублирования basename; standalone-режим работает; postMessage не падает в standalone.

## Часть D — Тесты (Vitest + Testing Library + MSW)
- Рендер ключевых компонентов с замоканным API (MSW): таблица позиций, календарь, панель сигналов.
- 401 → logout/redirect (этап 02).
- Флоатер в таблице показывает текущую доходность + пометку, не YTM.
- Форматтеры (деньги RUB, даты, дни до события) — юнит-тесты `utils`.
