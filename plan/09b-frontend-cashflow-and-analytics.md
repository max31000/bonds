> Часть 2 из 3 этапа 09 (декомпозиция — см. [`09-frontend.md`](09-frontend.md)). Самый крупный подэтап — основная графика продукта (§9 явно называет графику приоритетом продукта).

## Цель подэтапа

Сигналы, форс-обновление, настройки и интеграция с portal-shell — это 09c, **НЕ делать здесь**.

## B.2 — Календарь поступлений

Контракт (`GET /api/cashflow`, параметры `from`/`to` опциональны):
```json
{
  "byMonth": [{"month": "2026-07", "grossRub": 0, "taxRub": 0, "netRub": 0,
                "couponGrossRub": 0, "principalGrossRub": 0, "hasEstimatedFlows": false}],
  "byPosition": [{"positionId": 0, "instrumentId": 0, "grossRub": 0, "taxRub": 0, "netRub": 0, "hasEstimatedFlows": false}],
  "principalReleases": [{"date": "2026-07-01", "positionId": 0, "instrumentId": 0,
                          "flowType": "Amortization", "amountRub": 0, "isEstimated": false}],
  "disclaimer": ""
}
```

- Таймлайн/бары по месяцам (Recharts `BarChart`) — брутто/налог/нетто (стек или нетто с тултипом, раскладывающим брутто/налог).
- Отдельно — список/виджет дат освобождения тела (`principalReleases`) — когда появится ликвидность.
- Пометка `hasEstimatedFlows` — визуально (бумаги с оценочными потоками: флоатеры/неполные данные).
- Опционально — разбивка по позициям (`byPosition`) как вторичная таблица/раскрывающийся блок.

## B.3 — Scatter «дюрация × доходность»

Контракт (`GET /api/analytics/scatter`):
```json
{
  "points": [{"positionId": 0, "instrumentId": 0, "issuer": "", "modifiedDuration": 0,
              "effectiveYield": 0, "yieldKind": "Ytm", "isFloater": false, "isIndexed": false,
              "isEstimated": false, "dataIncomplete": false}],
  "curve": [{"termYears": 0.25, "yield": 0}],
  "curveAsOf": "2026-06-25",
  "disclaimer": ""
}
```

- Recharts `ScatterChart`: точки — позиции (X=дюрация, Y=доходность), линия — реконструированная кривая (`curve`).
- ⚠️ **Название кривой в UI** — нейтральное «безрисковая кривая» / «безрисковая доходность». **НЕ** использовать «КБД Московской Биржи» / «MOEX GCURVE» (товарный знак, §4.3) — касается только названия в интерфейсе, сами данные использовать можно.
- Визуальное различение точек: флоатер/индексируемая/неполные данные — другим цветом/маркером, с легендой.

## B.4 — Композиция портфеля

Контракт (`GET /api/analytics/composition`):
```json
{
  "totalMarketValueRub": 0,
  "byIssuer": [{"key": "", "marketValueRub": 0, "sharePercent": 0}],
  "bySector": [],
  "byCouponType": [],
  "byDurationBucket": [],
  "disclaimer": ""
}
```

- 4 виджета (по эмитенту/сектору/типу купона/корзинам дюрации) — пироги или горизонтальные бары (Recharts `PieChart`/`BarChart`), доли в %.
- Переключатель между 4 разрезами (таб/селект на одном виджете) либо 4 отдельных небольших виджета на странице — на твой выбор по месту, главное не загромождать экран.

## B.5 — Кривая доходности портфеля во времени

Контракт (`GET /api/analytics/xirr`, параметры `from`/`to` опциональны):
```json
{
  "currentXirr": 0,
  "history": [{"date": "2026-06-25", "marketValueRub": 0, "investedRub": 0, "xirr": 0}],
  "disclaimer": ""
}
```

- Текущий XIRR — крупное число/карточка.
- Recharts `LineChart` — динамика `marketValueRub`/`xirr` во времени по `history`.
- **История может быть пустой** (копится с первого запуска планировщика, этап 07) — это ожидаемо, не ошибка. Показать понятное состояние «недостаточно данных для графика, копится с первого синка», не падать и не рисовать пустой график без объяснения.

## Тесты (Vitest + Testing Library + MSW)

- Рендер каждого из 4 виджетов с замоканными ответами API (MSW), включая краевые случаи: пустая история XIRR, пустой календарь, scatter с флоатером/неполными данными.
- Проверка, что нигде в DOM не встречается текст «MOEX GCURVE»/«КБД Московской Биржи» (простая защита от регресса по товарному знаку).
- Форматтеры `utils`, которые использует эта часть (проценты, доли, агрегаты по месяцам) — юнит-тесты.

## Критерии приёмки

- [ ] Календарь поступлений: бары по месяцам (брутто/налог/нетто) + даты освобождения тела; оценочные потоки помечены.
- [ ] Scatter: точки позиций над линией безрисковой кривой; название в UI — «безрисковая кривая», не товарный знак; флоатеры/неполные данные визуально различимы.
- [ ] Композиция: 4 разреза (эмитент/сектор/тип купона/корзины дюрации), доли в %.
- [ ] Кривая XIRR: текущее значение + динамика во времени; пустая история — понятное состояние, не ошибка.
- [ ] Дисклеймер виден на каждом аналитическом экране (переиспользован компонент из 09a).
- [ ] `yarn typecheck && yarn test:run && yarn build` — без ошибок.
- [ ] Тесты 09a не сломаны.
