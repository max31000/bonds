## B.6 — Панель сигналов

Контракт:
- `GET /api/signals?unreadOnly=` → `{ "signals": [{"id":0,"type":"","severity":"High","positionId":0,"instrumentId":0,"suggestedAction":"","date":"2026-06-25","isRead":false}] }`
- `POST /api/signals/{id}/read` → `{"id":0,"isRead":true}`

- Список сигналов с акцентом по `severity` (высокая — например оферты — визуально выделена, цвет/иконка).
- Действие «отметить прочитанным» → `POST /api/signals/{id}/read`, обновление списка/счётчика непрочитанных.
- Опционально — счётчик непрочитанных в навигации (Mantine `Badge`).
- Используй `useSignalsStore` (Zustand, без persist) или локальный хук-обёртку над запросом — на твой выбор по месту, главное не плодить рассинхронизацию состояния.

## B.7 — Форс-обновление

Контракт:
- `POST /api/sync` → `{"alreadyRunning":false,"noAccountConfigured":false,"instrumentsSynced":0,"operationsUpserted":0,"yieldCurveUpdated":false,"positionsProjected":0,"flowsWritten":0,"snapshotStored":false,"signalsCreated":0,"errors":[],"hasErrors":false}`
- `GET /api/sync/status` → `{"isRunning":false,"lastRunStartedAtUtc":null,"lastSuccessAtUtc":null,"lastFailureAtUtc":null,"lastRunErrors":[]}`

- Кнопка «Обновить данные» — в навигации/шапке, доступна с любого экрана, не только тут.
- Индикатор состояния: дизейблить кнопку и показывать спиннер, если `isRunning`/`alreadyRunning`; периодически опрашивать `/sync/status`, пока идёт синк.
- Mantine `notifications` по завершении: успех (краткая сводка — сколько инструментов/операций обновлено) или ошибка (`hasErrors`/`errors[]`, без утечки секретов в сообщении).

## B.8 — Настройки

Контракт:
- `GET/PUT /api/settings` → `{"baseCurrency":"RUB","tInvestTokenConfigured":false,"tInvestTokenMasked":null,"upcomingEventDaysThreshold":0,"uninvestedCashThresholdRub":0,"uninvestedCashLookbackDays":0,"yieldBelowAlternativeBpsThreshold":0,"maturityWindowDaysForAlternativeComparison":0,"defaultMaxConcentrationPercent":0,"durationDriftToleranceYears":0}`
- `PUT /api/settings/tinvest-token` `{"token":""}` → `{"tInvestTokenConfigured":true,"tInvestTokenMasked":"...1234"}`

- Форма (Mantine `useForm`) для порогов триггеров (поля выше) — числовые инпуты с разумной валидацией (неотрицательные, разумные верхние границы).
- Поле ввода токена T-Invest: **обычное write-only поле** — после сохранения показывать только `tInvestTokenMasked`/статус «задан», саму форму очищать; токен из API никогда не приходит обратно — не пытайся его туда подставить или закэшировать в стейте дольше, чем на момент отправки.
- Тема (если уже заложена в `theme.ts`/`useSettingsStore` с `persist`) — переключатель светлая/тёмная, не обязательно, если не предусмотрено — не усложняй сверх необходимого.
- `TargetAllocation` (опциональная сущность по §5 спеки) — если есть смысл дать минимальный UI (список целевых долей по эмитентам/лимитов концентрации, добавление/удаление) — реализовать по минимуму; это не блокирующая часть.

## Часть C — Интеграция с portal-shell

- Проверить (или добавить, если ещё нет доступа) запись `bonds` в `portal-shell/registry.json` — **это отдельный репозиторий**; если доступа нет в этой среде, явно зафиксировать как ручной шаг (аналогично «Деплой — TODO» из этапа 01 в README), не выдумывать состояние.
- Пройти чеклист `portal-shell/docs/service-routing-integration.md` насколько это возможно без живого прода:
  - ассеты собираются с `/bonds/assets/...` (проверяется `yarn build` — уже было на этапе 01, перепроверить, что не регрессировало за 09a/09b);
  - внутренняя навигация даёт пути `/bonds/...` (basename работает);
  - F5 на вложенном роуте — клиентский роутинг, не серверный (на проде SPA fallback даёт nginx `try_files` из этапа 01 — здесь достаточно убедиться, что роутинг не предполагает серверной поддержки);
  - standalone-режим (`yarn dev`/`yarn preview`, без iframe) работает полностью;
  - postMessage (`useShellSync`) не падает в standalone (graceful no-op, если `window.parent === window`);
  - прод-проверки (`https://mvv42.ru/...`) — недостижимы без живого VDS (см. README «Деплой — TODO»), пометить как «не проверено» честно, не имитировать.

## Тесты

- Панель сигналов: рендер с MSW, отметка прочитанным обновляет UI.
- Форс-обновление: кнопка дизейблится во время `isRunning`, нотификация по завершении (успех/ошибка).
- Настройки: сохранение порогов; токен — после `PUT` поле очищается, токен не «эхо»-ится в UI; рендер формы с MSW.
- Полный прогон существующих тестов 09a/09b — не сломаны.
