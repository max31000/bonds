> **Кому адресовано.** Агент-исполнитель. Задача самодостаточна, почти целиком frontend. Общие
> конвенции — [CLAUDE.md](../CLAUDE.md); фронт: React 19 + TypeScript strict + Mantine 9 + Recharts 3,
> **yarn** (не npm), тесты vitest.
>
> **Зависимости:** мягкие. Интрадей-виджет берётся из задачи 16, живой XIRR — из 15; если они ещё не
> сделаны, дашборд собирается на существующих эндпоинтах, а места под их виджеты закладываются.
> Задачи 15/16/17 трогают те же страницы — при конфликте порядок: сначала они, потом эта.
>
> Перед пушем — `pre-push-check` на зелёном (typecheck — то, что чаще всего роняло CI).

## Проблема

1. Главная страница — сразу таблица позиций; нет «взгляда за 5 секунд»: сколько всего, что
   изменилось, что требует внимания.
2. Графики выглядят сыро, и все по-разному: подписи осей наезжают на легенду (классика Recharts —
   `label position: insideBottom, offset: -5` конфликтует с `<Legend/>`, видно в
   [Analytics.tsx](../bonds-web/src/pages/Analytics.tsx)), длинные суммы «1 234 567 ₽» на осях,
   у каждого виджета свои тултипы/отступы/цвета.
3. Scatter «Дюрация × доходность» непонятен пользователю — не хватает объяснения, зачем он.

## Что сделать

### A. Chart-kit — общая обвязка графиков (`bonds-web/src/components/charts/`)

Не обёртка-монстр над Recharts, а маленький набор соглашений:

1. `chartTheme.ts` — константы: палитра из Mantine CSS-переменных (переиспользовать `PIE_COLORS`
   из Analytics.tsx — вынести туда), стандартные margin'ы графика (с местом под подписи осей БЕЗ
   наезда на легенду: легенда `verticalAlign="top"`, подпись оси X — снизу), сетка
   (`strokeDasharray="3 3"` уже везде — закрепить).
2. `ChartTooltip.tsx` — один компонент тултипа (Paper + строки «метка: значение»), принимает
   форматтеры; заменить им пять самописных тултипов в Analytics.tsx.
3. `formatCompactRub` в [format.ts](../bonds-web/src/utils/format.ts): 1 234 567 → «1,23 млн ₽»,
   12 400 → «12,4 тыс ₽» (для осей и KPI; в тултипах — полные суммы). Юнит-тесты на пороги.
4. Ось дат: хелпер тиков — не больше ~8 подписей, наклон `angle={-30}` при тесноте.
5. `ChartCard.tsx` — Paper-обёртка виджета: заголовок, слот под контролы, **иконка «?»**
   (Popover) с объяснением графика — см. C.
6. Прогнать все существующие виджеты Analytics + Cashflow ([Cashflow.tsx](../bonds-web/src/pages/Cashflow.tsx))
   через chart-kit: одинаковые отступы, тултипы, компактные оси. Поведение/данные не менять —
   существующие тесты страниц должны остаться зелёными (правки тестов — только по вёрстке).

### B. Дашборд — новая главная (`bonds-web/src/pages/Dashboard.tsx`)

Роут `/`, таблица позиций переезжает на `/positions` (обновить NAV_ITEMS в
[AppLayout.tsx](../bonds-web/src/components/AppLayout.tsx), тесты роутинга в
[App.test.tsx](../bonds-web/src/App.test.tsx)).

Состав (всё — существующие данные):
1. **KPI-строка** (4 карточки): стоимость портфеля (+изменение за день, если задача 16 готова —
   из `/api/live/positions`; иначе скрыть дельту), XIRR (из `/api/analytics/xirr`), ближайшее
   поступление («12 июн — купон Х, 1 240 ₽» из `/api/cashflow`), непрочитанные сигналы (из
   существующего useSignalsStore, клик → /signals).
2. **График стоимости**: интрадей-виджет из задачи 16, если есть; иначе — история из
   `/api/analytics/xirr` (поле `marketValueRub`).
3. **Мини-композиция** по эмитентам (компактный donut, топ-5 + «прочие») — данные
   `/api/analytics/composition`.
4. **Ближайшие 30 дней** денежного потока — компактный список из `/api/cashflow`.
5. Пустые состояния каждой карточки — самостоятельные («нет данных — запустите синк»), дашборд не
   падает целиком из-за одного отказавшего эндпоинта (загрузка виджетов независимая).

### C. Объяснимость графиков

Тексты в Popover'ы `ChartCard` (по-русски, 2–4 предложения, без жаргона). Обязательные:
- **Дюрация × доходность (scatter):** «Каждая точка — ваша бумага. Серая линия — безрисковая
  кривая ОФЗ: доходность, которую дают гособлигации той же дюрации. Чем выше точка над линией —
  тем больше премия за кредитный риск. Бумага заметно ниже линии при той же дюрации — кандидат на
  замену (см. Рекомендации)». Плюс на самом графике: подписывать точки именем бумаги
  (`LabelList`/кастомный shape, обрезать до ~12 символов), при наведении — полное имя.
- **XIRR**, **Сценарии ставок**, **Траектория**, **Композиция** — по одному абзацу: что считается,
  от какой базы, что означает движение вверх/вниз.

### D. Тесты

Vitest: formatCompactRub (пороги «тыс/млн», отрицательные); дашборд — рендер KPI из MSW-данных,
независимость виджетов (один эндпоинт 500 → остальные карточки живы); роут `/positions` работает;
Popover с объяснением открывается.

## Критерии приёмки

- [ ] Все графики Analytics/Cashflow используют chart-kit: ничего не наезжает, оси компактные,
      тултипы единообразные.
- [ ] `/` — дашборд с KPI, графиком стоимости, композицией и ближайшими поступлениями;
      таблица — на `/positions`.
- [ ] У каждого графика — «?» с человеческим объяснением; точки scatter подписаны.
- [ ] Отказ одного эндпоинта не роняет дашборд.
- [ ] `yarn typecheck`/`test:run`/`build` зелёные; `pre-push-check` пройден.
