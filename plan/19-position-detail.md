## Проблема

Таблица позиций — единственное представление бумаги. Некуда «провалиться»: посмотреть график цены,
календарь купонов именно этой бумаги, свои операции по ней, полный набор метрик с объяснениями,
прикинуть «что получу, если продам сейчас».

Бэкенд уже отдаёт `GET /api/positions/{id}`
([PositionsEndpoints.cs](../src/Bonds.Api/Endpoints/PositionsEndpoints.cs)) — проверь состав ответа
и расширь его недостающим (см. ниже), а не плоди параллельный эндпоинт.

## Что сделать

### A. Backend — обогатить `GET /api/positions/{id}`

Добавить к существующему ответу:
1. **`priceHistory`** — дневные цены за выбранный период (`?range=1m|6m|1y|all`): из
   MOEX history-клиента (`GetHistoryPricesAsync`, задача 15 §A), кэшировать в таблице
   `instrument_price_history` (миграция; upsert по (instrument, date); дозагружать только
   недостающий хвост, не всю историю каждый раз).
2. **`couponSchedule`** — прошедшие и будущие купоны бумаги с датами и суммами (данные уже в БД:
   [CouponScheduleRepository.cs](../src/Bonds.Infrastructure/Repositories/CouponScheduleRepository.cs));
   аналогично амортизации и оферты
   ([AmortizationScheduleRepository](../src/Bonds.Infrastructure/Repositories/AmortizationScheduleRepository.cs),
   [OfferScheduleRepository](../src/Bonds.Infrastructure/Repositories/OfferScheduleRepository.cs)).
3. **`operations`** — журнал операций пользователя по этому инструменту (покупки/продажи/купоны/
   комиссии, из [OperationRepository](../src/Bonds.Infrastructure/Repositories/OperationRepository.cs)).
4. **`ifSoldNow`** — оценка «продать сейчас»: рыночная стоимость − комиссия
   (`SwitchAnalysisService.DefaultCommissionRate`) + НКД, и (если задача 14 сделана) итоговый P&L
   с купонами. Чистый расчёт — рядом с cost-basis-сервисом, с юнит-тестом.
5. Интеграционный тест на состав ответа с засеянными графиками/операциями.

### B. Frontend — страница `/positions/:id`

Новая страница `bonds-web/src/pages/PositionDetail.tsx`, роут, переход — клик по строке таблицы
[Positions.tsx](../bonds-web/src/pages/Positions.tsx) (строку сделать кликабельной целиком,
`cursor: pointer`).

Состав сверху вниз:
1. **Шапка**: имя, ISIN, эмитент/сектор, бейджи (флоатер/оферта/неполные данные — те же, что в
   таблице), количество × текущая цена = стоимость.
2. **График цены** (line/area, переключатель 1м/6м/1г/всё). Поверх — маркеры моих покупок/продаж
   (ReferenceDot по датам операций) и, если есть данные задачи 14, горизонтальная линия средней
   цены входа (ReferenceLine с подписью).
3. **Метрики** — грид карточек: YTM/доходность к оферте, текущая доходность, дюрация (мод.),
   G-спред, НКД, следующий купон. У каждой — короткий tooltip-глоссарий (2 предложения).
4. **Календарь бумаги**: таблица будущих купонов/амортизаций/оферт с датами и суммами на мою
   позицию (сумма купона × количество), прошедшие — свёрнуты.
5. **Мои операции** — таблица журнала по бумаге.
6. **«Если продать сейчас»** — карточка с оценкой из `ifSoldNow` + дисклеймер (существующий
   [Disclaimer.tsx](../bonds-web/src/components/Disclaimer.tsx)).

### C. Тесты

Vitest + MSW: рендер всех секций по мок-ответу; клик по строке таблицы ведёт на карточку;
переключение range перезапрашивает историю; пустая история цен → плейсхолдер, страница не падает.

## Критерии приёмки

- [ ] Клик по позиции открывает карточку с графиком цены, метриками, календарём, операциями.
- [ ] История цен кэшируется в БД, догружается только хвост.
- [ ] «Если продать сейчас» считает выручку за вычетом комиссии (+P&L при наличии задачи 14).
- [ ] `dotnet test`, `yarn typecheck`/`test:run`/`build` зелёные; `pre-push-check` пройден.
