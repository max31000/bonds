## Проблема

Сервис выглядит «максимально скучно и просто»: одна светлая тема, плоская таблица без визуальных
акцентов, на телефоне таблица на 10+ колонок нечитаема (только горизонтальный скролл).

## Что сделать

### A. Тёмная тема

1. Mantine 9 поддерживает color scheme из коробки: `defaultColorScheme="auto"` на
   `MantineProvider` ([App.tsx](../bonds-web/src/App.tsx), тема — [theme.ts](../bonds-web/src/theme.ts)).
   Переключатель светлая/тёмная/системная — `ActionIcon` в шапке
   ([AppLayout.tsx](../bonds-web/src/components/AppLayout.tsx)), состояние Mantine персистит сам
   (localStorage).
2. Пройтись по местам с захардкоженными цветами: графики уже на CSS-переменных
   (`var(--mantine-color-...)`) — проверить контраст сетки/тултипов в тёмной теме (тултипы —
   `Paper`, должны переехать сами); семантические цвета P&L (зелёный/красный) — брать из темы, не
   литералами.
3. Сервис живёт в iframe portal-shell ([useIsInsideShell.ts](../bonds-web/src/components/useIsInsideShell.ts),
   [useShellSync.ts](../bonds-web/src/components/useShellSync.ts)) — проверь, не передаёт ли шелл
   тему; если нет — просто не конфликтовать (auto по prefers-color-scheme достаточно).

### B. Таблица позиций — визуальные акценты

[Positions.tsx](../bonds-web/src/pages/Positions.tsx):
1. Heatmap-подсветка колонки «Доходность»: фон ячейки от бледного к насыщенному по перцентилю
   внутри портфеля (аккуратные alpha-оттенки одной палитры, работающие в обеих темах;
   floater/indexed — не красить, их доходность несравнима).
2. P&L (если задача 14 сделана): стрелка ▲/▼ + цвет.
3. Строка «Итого» снизу: суммарная стоимость, средневзвешенная доходность и дюрация портфеля
   (веса — рыночная стоимость; floater'ы в средневзвешенную доходность не включать, добавить
   сноску «* без флоатеров»).

### C. Мобильная адаптация (< `sm`)

1. [AppLayout.tsx](../bonds-web/src/components/AppLayout.tsx): navbar уже с
   `breakpoint: 'sm'`, но burger-кнопки нет — навигация на мобиле недоступна. Добавить
   `Burger` + `AppShell.Navbar` collapse (стандартный паттерн Mantine AppShell).
2. Таблица позиций: при `useMediaQuery('(max-width: em(768))')` рендерить карточки вместо
   `<Table>` — компонент `PositionCard.tsx`: имя + бейджи, стоимость, доходность, P&L; остальные
   метрики — в раскрывашке (`Collapse`). Сортировка — `Select` над списком.
3. Дашборд (задача 18): KPI-карточки в одну колонку, графики на всю ширину (Mantine `SimpleGrid`
   с `cols={{ base: 1, md: 2 }}` — вероятно уже так; проверить фактически на 375px).
4. Графики: на узких экранах меньше тиков оси X (хелпер из chart-kit), высота 240 вместо 300–360.
5. Проверка руками через vite dev + браузерный эмулятор 375px: все страницы без горизонтального
   скролла (кроме осознанного скролла внутри таблиц там, где карточки не применимы, например
   календарь Cashflow).

### D. Тесты

Vitest: переключатель темы меняет colorScheme; таблица→карточки по media query (мокнуть
`matchMedia`); строка «Итого» считает средневзвешенные значения (юнит на функцию агрегации,
вынести её в `utils`).

## Критерии приёмки

- [ ] Тёмная/светлая/auto переключается из шапки и переживает перезагрузку; графики и таблицы
      читаемы в обеих темах.
- [ ] Доходность в таблице подсвечена heatmap'ом; есть строка «Итого» со средневзвешенными.
- [ ] На 375px: рабочая навигация (burger), карточки позиций, дашборд в одну колонку, без
      горизонтального скролла страницы.
- [ ] `yarn typecheck`/`test:run`/`build` зелёные; `pre-push-check` пройден.
