## Проблема

Пользовательский сценарий-мечта: слева — слабая позиция портфеля, справа — выпадашка со всем
рынком: по умолчанию топ-10 самых доходных (после гигиенических фильтров банка), поиск «газпром» →
топ-10 Газпрома по доходности. Выбрал бумагу → мгновенная карточка сравнения «держать vs
переложиться» с полной формулой (спред, комиссия из твоего тарифа, НКД, налог — всё уже готово).

## Что сделать

### A. Материализация бумаги из банка (backend)

Банк хранит только статистику MOEX; для точного сравнения бумагу надо прогнать через наш движок.

1. `POST /api/universe/{secid}/materialize` → `{ instrumentId, metrics: {...} }`:
   - резолвит ISIN из `bond_universe`;
   - заводит/находит Instrument тем же путём, что watchlist:
     `BondSyncService.ResolveOrCreateInstrumentByIsinAsync` + котировка через логику
     [WatchlistSyncService.cs](../src/Bonds.Infrastructure/Sync/WatchlistSyncService.cs) —
     **вытащи общий кусок «обогатить одну бумагу по ISIN» в переиспользуемый метод/сервис**
     (например `InstrumentEnrichmentService`), чтобы watchlist и материализация не дублировали
     код; watchlist-запись при этом НЕ создаётся;
   - возвращает instrumentId + полные метрики движка (`PortfolioHoldingsBuilder.BuildForInstrumentsAsync`,
     как это делают watchlist/матрица): точный YTM, дюрация, G-спред, dataIncomplete-флаги.
   - Бумага не нашлась на MOEX / не облигация / нет данных → 422 с человекочитаемой причиной.
   - Идемпотентно: повторный вызов не плодит дублей, обновляет котировку.
2. Интеграционные тесты (MOEX замокан): materialize нового secid → instrument создан, метрики
   посчитаны; повторный вызов идемпотентен; несуществующий secid → 422; watchlist НЕ пополнился.

### B. Выпадашка-сравнивалка (frontend)

Секция «Слабые звенья» в [Recommendations.tsx](../bonds-web/src/pages/Recommendations.tsx):

1. У каждой карточки слабого кандидата — кнопка/область «Сравнить с рынком» → раскрывается
   компонент `MarketComparator` (новый, `bonds-web/src/components/MarketComparator.tsx` —
   отдельный файл, страница уже большая):
   - Mantine `Select`/`Combobox` с серверным поиском: пусто → `GET /api/universe?sortBy=yield&limit=10`
     (топ-10 доходных, скрытые исключены); ввод текста → тот же запрос с `search=` (debounce
     ~300мс). В опции: имя, YTM %, дюрация, бейдж ликвидности; уже в портфеле/watchlist — пометка.
   - Выбор бумаги → `POST /api/universe/{secid}/materialize` → затем
     `POST /api/analytics/replacement` (`holdPositionId` слабой позиции + `targetInstrumentId`) →
     карточка результата в ТОМ ЖЕ формате разбивки-формулы, что у матрицы замен (задача 23):
     спред → капитал → горизонт → валовая → − комиссии → − НДФЛ → чистая выгода ≈ % годовых.
     Переиспользуй существующую разметку разбивки — если она вшита в карточку матрицы, вытащи в
     общий компонент, не копипасть.
   - Лоадеры на оба шага, ошибки — Alert с текстом от бэкенда (422-причина).
   - Кнопка «В watchlist» на карточке результата (существующий `POST /api/watchlist`).
2. Не ломай существующие секции (матрица замен, аллокация, watchlist, «вне сравнения»).
3. Дисклеймер: сравнение с рыночной бумагой использует данные MOEX; ликвидность и реальная цена
   исполнения могут отличаться (упомяни slippageEstimate из банка, если ликвидность Low — покажи
   предупреждение прямо на карточке).

### C. Тесты фронта

Vitest + MSW: пустой ввод → топ-10; поиск фильтрует; выбор → materialize → replacement → карточка
с формулой; 422 от materialize → ошибка в UI; предупреждение о низкой ликвидности; кнопка
watchlist дергает POST.
