## Проблема

Банк облигаций есть, но исследовать его можно только через выпадашку у слабых позиций. Нужна
полноценная страница-скринер: весь рынок таблицей с фильтрами, сортировками, поиском и действием
«в watchlist» — как у Rusbonds/Cbonds, но со своими метриками ликвидности и G-спреда.

## Что сделать

### A. Страница

1. Новая страница `bonds-web/src/pages/Screener.tsx`, роут `/screener`, пункт навигации
   «Скринер» в [AppLayout.tsx](../bonds-web/src/components/AppLayout.tsx) (после «Рекомендации»;
   НЕ сломай индикатор синка/burger/переключатель темы).
2. Store `useScreenerStore.ts` + api-клиент `api/universe.ts` (если 27 уже создала — расширь).
3. Состав страницы:
   - **Строка статуса**: «В банке N бумаг · скрыто M (неликвид/листинг/аномальная доходность) ·
     обновлено HH:MM» — из `GET /api/universe/status` + `hiddenCount`; переключатель «показать
     скрытые» (`includeHidden=true`, скрытые строки — приглушённые, с бейджем причины).
   - **Панель фильтров** (Mantine, компактно, сворачиваемая на мобиле): поиск (debounce),
     диапазон YTM (%), диапазон дюрации (лет), сектор (Select из данных), уровень листинга,
     минимальная ликвидность (скор). Фильтры — серверные (параметры `GET /api/universe`).
   - **Таблица** (серверная пагинация по 50, сортировка кликом по заголовку — yield/duration/
     turnover/gspread): Бумага (имя + ISIN tooltip) · Сектор · YTM · Дюрация, лет · Цена, % ·
     G-спред (прибл.), б.п. · Оборот/день (компактный формат) · Ликвидность (бейдж-скор) ·
     Погашение/оферта · Действия. Формат чисел — существующие форматтеры
     ([format.ts](../bonds-web/src/utils/format.ts)); G-спред «прибл.» — с tooltip «оценка по
     данным MOEX относительно кривой ОФЗ, не точный расчёт движка».
   - **Метки принадлежности**: бейджи «в портфеле» / «в watchlist» на строках (флаги уже в API).
   - **Действия строки**: «+ watchlist» (существующий `POST /api/watchlist`, оптимистично,
     после — бейдж меняется); «сравнить» — если задача 27 влита, переход на /recommendations
     не делай — просто скопируй в буфер ISIN не надо — оставь ТОЛЬКО «+ watchlist», сравнение
     живёт в рекомендациях (не плоди вторую точку входа в сравнение, YAGNI).
4. Заголовок страницы — с «?»-пояснением (паттерн ChartExplainIcon): что за банк, откуда данные,
   что значит «скрытые», дисклеймер «данные MOEX, не рекомендация».

### B. Мобильная версия

Таблица → карточки (паттерн `PositionCard` из задачи 21): имя+YTM крупно, дюрация/ликвидность/
цена в строку, остальное в Collapse. Панель фильтров — в Drawer/Collapse. Пагинация «показать ещё».

### C. Тесты

Vitest + MSW: рендер таблицы из мок-ответа; фильтр меняет запрос (параметры); сортировка кликом;
«показать скрытые» подсвечивает причину; «+ watchlist» дергает POST и меняет бейдж; статусная
строка; мобильные карточки (matchMedia замокан). Бэкенд не трогаешь — но `dotnet build/test`
прогони на всякий случай.

## Критерии приёмки

- [ ] `/screener` в навигации; фильтры/поиск/сортировка/пагинация — серверные, через существующий API.
- [ ] Скрытые бумаги показываются по переключателю с причиной; статусная строка честная.
- [ ] «+ watchlist» работает с оптимистичным обновлением; метки «в портфеле/watchlist» видны.
- [ ] Мобильная версия — карточки; ничего в AppLayout не сломано.
- [ ] `yarn typecheck/test:run/build/lint` + `dotnet build/test` — зелёные.
