## Проблема (полевой фидбек владельца)

1. **P1-5:** в блоке 1 клик по кандидату из списка (~20 строк, режимы «доходные рынка» /
   «дешёвые соседи») рендерит карточку выгоды ПОД всем списком — вне вьюпорта; выглядит как
   «ничего не произошло»/баг.
2. **P1-6:** в списках кандидатов нет фильтра по дюрации (заменять 2-летнюю бумагу 7-летней —
   несравнимый риск) и не видно даты оферты (критично: доходность бумаги с офертой считается к
   оферте, горизонт совсем другой).

## Что сделать

### A. Бэкенд — аддитивные поля DTO (минимум)

1. `ReplacementCandidateDto` ([AnalyticsEndpoints.cs](../src/Bonds.Api/Endpoints/AnalyticsEndpoints.cs),
   задача 33) — добавить `DateOnly? OfferDate` из `BondUniverseEntry.OfferDate` (банк уже хранит,
   [BondUniverseEntry.cs:37](../src/Bonds.Core/Models/BondUniverseEntry.cs)). Для mode=rv поле
   бери из той же банк-записи по secid (как берутся yieldFraction и др.).
2. `UniverseRowDto` ([UniverseEndpoints.cs](../src/Bonds.Api/Endpoints/UniverseEndpoints.cs)) —
   проверь, есть ли `OfferDate`; если нет — добавить (нужен сравнивалке `MarketComparator`,
   которая строит опции из universe-строк). Сериализацию `DateOnly` сверь с тем, как соседние
   даты уже ходят в JSON (`MaturityDate` где-нибудь в DTO) — единый формат.
3. Интеграционный тест: поле присутствует и корректно для посеянной бумаги с офертой (оба DTO).

### B. Фронт — P1-5: карточка выгоды у выбранной строки

1. В списке кандидатов блока 1 (`Recommendations.tsx`, панель режимов из T-35): карточку выгоды
   (`ReplacementBreakdown`-обёртку) рендерить **непосредственно под выбранной строкой** списка
   (выбранная строка подсвечена, карточка раскрывается под ней), а не после всего списка.
   Повторный клик по той же строке — свернуть. Клик по другой — карточка переезжает под неё.
2. После раскрытия — `scrollIntoView({ block: 'nearest', behavior: 'smooth' })` на карточку
   (на случай, если строка у нижнего края). Учти паттерн Mantine `Collapse` из гайда (в тестах
   проверять `toBeVisible()` после раскрытия).
3. В `MarketComparator` (режим «поиск») карточка уже рендерится сразу под селектом — там только
   добавить тот же `scrollIntoView` после появления результата, если карточка ниже вьюпорта.

### C. Фронт — P1-6: фильтр дюрации + дата оферты

1. Типы: `ReplacementCandidate.offerDate?: string | null`, `UniverseRow.offerDate?: string | null`
   (зеркало A; формат сверь с фактической сериализацией).
2. В строке кандидата (режимы market/rv) показывать: дюрацию (уже есть `durationYears` — если не
   отображается, показать) и бейдж «оферта {дата}» при наличии `offerDate` (формат дат — как в
   соседних местах `format.ts`). В опциях/карточке `MarketComparator` — тот же бейдж оферты.
3. Фильтр дюрации в панели кандидатов блока 1: `Switch`/`Chip` **«Похожая дюрация (±1.5 года)»**,
   по умолчанию ВЫКЛ. Окно — вокруг `modifiedDuration` заменяемой позиции (она есть в строке
   сравнения `ComparisonRow.modifiedDuration`); константа 1.5 — та же, что окно матрицы замен
   (`ComparableDurationWindowYears`, см. [ReplacementMatrixService.cs:65](../src/Bonds.Core/Analytics/ReplacementMatrixService.cs))
   — продублируй значение констан­той на фронте с комментарием-ссылкой. Фильтрация клиентская по
   уже загруженному списку; кандидаты с `durationYears == null` при включённом фильтре скрываются.
   Показать «осталось N из M» рядом с переключателем, чтобы усечение не выглядело багом.
4. Состояния: включённый фильтр, под которым не осталось кандидатов → понятная пустая заглушка
   («нет кандидатов с похожей дюрацией — отключите фильтр»), не пустой экран.

### D. Vitest-кейсы

1. Клик по кандидату → карточка выгоды видима сразу под выбранной строкой (не в конце списка);
   повторный клик сворачивает; клик по другой строке переносит карточку.
2. Фильтр «похожая дюрация»: список режется по окну ±1.5 г от дюрации позиции; счётчик «N из M»;
   кандидаты без дюрации скрыты при включённом фильтре; пустое состояние с подсказкой.
3. Бейдж «оферта {дата}» рендерится при `offerDate`, отсутствует при `null`.
4. Обновлённые msw-моки несут `offerDate` (и в universe-строках для `MarketComparator`).

- [ ] Дата оферты видна в кандидатах обоих режимов и в сравнивалке; бэкенд-поля аддитивны и
      покрыты интеграционным тестом.
