> **Рамки (что НЕ делать):**

## Проблема (запрос владельца)

Владелец при замене бумаг не может быстро отличить «нормальный» кандидат от мусорного: два
отдельных сигнала (ликвидность, спред) есть, но нет ОДНОГО агрегата и фильтра по нему.
Платных рейтингов нет и не будет в ближайшее время.

## Что сделать

### A. Агрегат в `CandidateRiskSignalService` (Bonds.Core)

1. Метод `Aggregate(CandidateRiskSignals signals, int? listLevel, string? sector) →
   ReliabilityLight { Green, Yellow, Red }` + строка-обоснование (какие сигналы притянули вниз).
   Матрица (зафиксировать в doc-comment и тестах):
   - **Red**: любой Caution (ликвидность Low/None+листинг-3 ИЛИ спред заметно выше медианы);
   - **Green**: обе оси Good/Neutral, ни одного Caution, листинг 1–2, спред не выше медианы
     более чем на порог, ликвидность не None; гос/муниципальный сектор → Green при любом
     спреде (кредит суверенный);
   - **Yellow**: всё остальное (смешанные сигналы, нехватка данных).
   - Нехватка данных (null-спред И null-ликвидность одновременно) → Yellow, не Red.
2. Расширить `RiskSignalsDto` полями `reliability: "Green"|"Yellow"|"Red"` и
   `reliabilityReason: string`. Аддитивно — существующие поля не менять.
3. Прокинуть агрегат во все места, где уже считаются риск-сигналы: `replacement-candidates`
   (оба режима), `POST /replacement` (targetRiskSignals), аллокация market/recommended.
   Единая точка — там, где сейчас вызывается `AssessEntryRiskSignals`.

### B. Фильтр-светофор

1. `GET /api/analytics/replacement-candidates`: параметр `reliability={green|yellow|red}` —
   опциональный фильтр «не хуже уровня» (green = только Green; yellow = Green+Yellow; по
   умолчанию — без фильтра).
2. Скринер `GET /api/universe`: светофор в `UniverseRowDto` (по банк-записи; для бумаг без
   G-спреда — по остальным осям) + тот же параметр фильтра. In-memory, как остальные фильтры.

### C. Фронт

1. Компонент светофора: цветная точка/бейдж (Green зелёный / Yellow жёлтый / Red красный) с
   тултипом: reliabilityReason + оба детальных сигнала + дисклеймер «оценка по биржевой
   статистике, не кредитный рейтинг». Переиспользовать/расширить `RiskSignalBadges.tsx` —
   светофор слева, детальные бейджи в раскрытии/тултипе.
2. Светофор в: строках кандидатов блока 1, строках аллокации блока 2, скринере (колонка),
   карточке выгоды сравнивалки.
3. Фильтр в панели кандидатов блока 1 («не хуже: 🟢/🟡/все», дефолт — все) и в FilterPanel
   скринера. Честный счётчик «N из M» при активном фильтре (паттерн T-37).
4. Типы зеркалят DTO; msw-моки обновить.

### D. Тесты

1. Юнит: полная матрица агрегации (все комбинации осей × листинг × сектор × null-данные),
   эталоны руками.
2. Интеграционные: фильтр reliability в replacement-candidates и universe; светофор в
   аллокации.
3. Vitest: светофор рендерится с тултипом и дисклеймером; фильтры режут списки с счётчиком;
   пустые состояния.

## Критерии приёмки

- [ ] Один агрегированный светофор виден в кандидатах, аллокации, скринере, сравнивалке;
      детальные сигналы доступны в тултипе/раскрытии.
- [ ] Фильтр «не хуже уровня» работает в блоке 1 и скринере, с честным счётчиком.
- [ ] Светофор нигде не назван рейтингом; дисклеймер на месте; матрица агрегации
      задокументирована и закреплена тестами (включая null-данные → Yellow).
- [ ] Существующие бейджи/поля не сломаны (аддитивность DTO).
- [ ] Существующие тесты не ослаблены; `scripts/pre-push-check.sh --all` — зелёный.
